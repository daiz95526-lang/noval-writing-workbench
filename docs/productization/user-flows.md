# NOVAL Phase 1 用户流程

状态：待用户确认  
依赖：[01-product-definition.md](01-product-definition.md)、[02-information-architecture.md](02-information-architecture.md)

## 1. 流程设计约定

### 1.1 步骤标记

| 标记 | 含义 |
| --- | --- |
| 必须 | 不完成就不能进入依赖该数据的后续步骤 |
| 可跳过 | 不阻塞主流程，但产品应解释影响 |
| 需确认 | AI 或系统不能替用户做最终决定 |
| 自动 | 系统应在满足条件后自动执行或保存 |
| 可恢复 | 失败后保留输入、稳定版本和部分结果 |

### 1.2 通用恢复原则

1. 用户输入先保存为草稿，再启动长任务。
2. 长任务开始后立即获得可读名称和所属项目，不向用户展示 task ID。
3. 页面关闭或切换项目不取消任务。
4. 已完成的阶段和部分结果持续保存。
5. 重试默认复用原输入，用户可以先修改再重试。
6. 失败消息包含原因类别、影响、可执行下一步和“查看详情”。
7. 任何 AI 结果都先进入候选状态，不自动覆盖正式内容。

## 2. 完整核心流程

```text
创建项目
→ 配置模型
→ 导入语料
→ 扫描章节
→ 查看分析
→ 自动构想
→ 审核总体构想
→ 生成完整章节规划
→ 选择章节
→ 生成完整章节
→ 人工审核
→ 迭代修改
→ AI 深度质检
→ 正式保存
→ 下一章
→ 导出
```

### 2.1 分步规则

| 步骤 | 规则 | 自动行为 | 用户确认 | 失败恢复 |
| --- | --- | --- | --- | --- |
| 1. 创建项目 | 必须 | 生成稳定项目标识、manifest 和空目录结构 | 确认项目名称、类型和位置 | 表单保留；不完整目录标记为创建失败并可安全清理 |
| 2. 配置模型 | AI 功能前必须；项目创建时可跳过 | 继承全局默认并进行非敏感配置校验 | 用户保存配置并触发连接测试 | 保存非密钥输入；说明认证/网络/模型不兼容；允许离线浏览项目 |
| 3. 导入语料 | 续写/分析项目必须；原创项目可跳过 | 复制或引用前先校验格式、权限和重复 | 确认导入方式和只读来源 | 失败文件单独列出；成功文件保留；可修复后继续 |
| 4. 扫描章节 | 有语料时必须 | 导入后自动建议或自动开始；生成章节索引和报告 | 大规模覆盖旧索引时需确认 | 保留旧索引；新结果先写临时区；成功后原子替换 |
| 5. 查看分析 | 可跳过，强烈推荐 | 扫描完成后给出推荐分析项 | 用户决定分析全部/部分/稍后 | 逐章保存结果；失败章可单独重试，不清空成功结果 |
| 6. 自动构想 | 可跳过；可手工创建构想 | 汇总已选语料、知识和用户方向 | 用户确认使用哪些上下文 | 保留方向输入和原构想；重试生成新候选，不覆盖 |
| 7. 审核总体构想 | 生成规划前必须 | 自动保存编辑稿和比较版本 | 必须明确“确认总体构想” | 编辑稿可恢复；拒绝候选后回到修改/重新生成 |
| 8. 生成完整章节规划 | 推荐；也可手工建立 | 分批生成、持续保存已完成章节计划 | 覆盖已有规划前确认；最终计划可逐章编辑 | 保存部分规划；失败批次单独重试；旧规划保持可恢复 |
| 9. 选择章节 | 必须 | 默认推荐第一个未完成且依赖满足的章节 | 用户可改选，但跨越依赖时提示 | 选择状态本地与项目双重保存 |
| 10. 生成完整章节 | 可跳过（允许手写） | 分段生成、阶段保存、规则检查 | 启动前确认计划和目标字数；不需每段确认 | 超时保留已生成段落；可继续、重试当前段或接受部分稿 |
| 11. 人工审核 | 正式保存前必须 | 自动保存编辑内容并显示计划对照 | 用户决定接受、编辑、丢弃或另存候选 | 页面刷新恢复未提交草稿；不覆盖前一版本 |
| 12. 迭代修改 | 可跳过 | 对修改前版本自动建快照；生成修改候选 | 用户选择局部修改/整章重写并接受差异 | 原稿永远保留；失败候选可删除或重试 |
| 13. AI 深度质检 | 可跳过，正式保存前推荐 | 先执行规则检查，再执行 AI 检查 | 用户选择采纳哪些建议；AI 不自动改正式稿 | 质检失败不影响正文；可重试或仅使用规则结果 |
| 14. 正式保存 | 必须且需确认 | 创建版本、原子写入正式章节、更新计划状态 | 必须确认保存目标项目和章节 | 任一步失败回滚到原正式版本；候选稿继续保留 |
| 15. 下一章 | 可选循环 | 推荐下一未完成章节并带入连续性上下文 | 用户确认进入下一章 | 当前章保存状态不受影响，可稍后继续 |
| 16. 导出 | 可选 | 生成导出快照和校验文件 | 用户选择范围、格式和位置 | 失败不影响正式章节；保留导出参数并允许重试 |

## 3. 场景 A：第一次使用

### 3.1 主路径

```text
启动应用
→ 欢迎页面
→ 选择“创建第一个项目”
→ 输入项目名称
→ 选择项目类型：原创 / 续写 / 分析
→ 配置模型或选择“稍后配置”
→ 选择开始方式
   ├─ 导入自己的合法语料
   ├─ 使用原创示例
   └─ 创建空项目
→ 扫描与校验
→ 项目概览
→ 执行推荐下一步
```

### 3.2 页面要求

- 第一次进入不展示 0 章统计面板。
- 不出现 `longzu6`、固定章节数、固定目录或第三方模型内部名称。
- 欢迎页先说明数据保存在本地，以及调用远程模型时会发送哪些内容。
- 模型配置允许跳过，但所有 AI 主按钮应说明“需要先配置模型”，并直接链接到设置。
- 提供完全原创、可公开的最小示例，不依赖私有语料。
- 每一步只有一个明确主按钮；高级设置折叠。

### 3.3 中断恢复

- 向导每一步保存进度。
- 下次启动回到未完成步骤，而不是重新开始。
- 创建了项目但未导入语料时，项目显示“待准备”，不视为错误。
- 导入失败时可以返回选择其他文件，不删除已创建项目。

## 4. 场景 B：续写现有作品

### 4.1 主路径

```text
项目列表
→ 打开续写项目
→ 项目概览检查准备状态
→ 语料库确认章节与原作结尾
→ 分析中心查看风格和知识库
→ 创作 / 总体构想
→ 输入可选方向
→ 生成构想候选
→ 编辑并确认
→ 生成完整章节规划
→ 审核计划
→ 选择第一个未完成章节
→ 生成完整章节
→ 人工审核和修改
→ AI 质检
→ 确认保存为正式章节
→ 进入下一章
```

### 4.2 决策点

- 语料不足：允许继续，但明确“风格和设定一致性可能下降”。
- 分析未完成：允许先规划，生成时显示缺失上下文。
- 总体构想未确认：不允许自动批量生成完整章节规划。
- 当前章节前置章未完成：允许高级用户继续，但需要确认。
- 质检存在严重问题：允许正式保存，但必须二次确认并记录提示。

### 4.3 循环

每章完成后，项目概览和规划页同步更新：

```text
正式章节 N
→ 更新章节状态
→ 刷新连续性上下文
→ 推荐章节 N+1
→ 用户确认
→ 新章节工作区
```

## 5. 场景 C：已有草稿继续创作

### 5.1 主路径

```text
打开项目
→ 章节库
→ 选择候选稿或正式章节
→ 打开章节编辑器
→ 自动创建/确认版本基点
→ 选择操作
   ├─ 从末尾继续
   ├─ 局部修改
   ├─ 整章重写
   └─ 仅人工编辑
→ 比较候选结果
→ 接受到编辑稿
→ 规则检查 / AI 质检
→ 保存候选版本或更新正式章节
→ 导出
```

### 5.2 内容保护

- AI 操作从当前稳定版本创建新候选，不原地覆盖。
- 用户可以逐次接受或拒绝修改建议。
- 自动保存与正式保存是两个状态。
- 更新正式章节前展示章节、版本和修改摘要。
- 导出来自选定快照，导出失败不改变正文。

## 6. 场景 D：分析作品

### 6.1 主路径

```text
创建“分析”项目
→ 导入合法语料
→ 扫描章节
→ 查看导入报告和异常
→ 选择分析范围
→ 风格分析
→ 构建知识库
→ 浏览人物、地点、设定、主题和情节
→ 修正或补充分析结果（后续能力）
→ 将结果用于创作或导出
```

### 6.2 分析任务策略

- 支持分析单章、选中章节和全量语料。
- 全量分析先估算范围和模型调用影响。
- 逐章持久化；部分失败不使整个任务无结果。
- 知识库显示来源范围、生成时间和是否过期。
- 语料变化后，受影响分析标记为“需要更新”，不静默覆盖。

## 7. 章节工作区流程

### 7.1 稳定状态

```text
计划摘要
→ 编辑稿基线
→ AI 候选稿
→ 用户接受后的编辑稿
→ 质检报告
→ 正式章节版本
```

### 7.2 生成流程

1. 用户确认当前章节和计划摘要。
2. 系统自动保存目标字数、补充要求和已有编辑稿。
3. 系统开始分段生成并持续保存部分结果。
4. 任务完成后运行规则完整性检查。
5. 候选稿与计划并排展示。
6. 用户选择接受到编辑器、保留为候选或删除。

### 7.3 修改流程

1. 用户选中修改范围或整章。
2. 输入修改目标，系统保存原版本。
3. AI 返回修改候选和修改摘要。
4. 用户查看差异。
5. 用户接受、部分接受或拒绝。
6. 接受后形成新的编辑版本，不直接成为正式章节。

### 7.4 质检流程

1. 自动规则检查：章节长度、结尾完整性、计划节点覆盖等。
2. 用户可启动 AI 深度质检。
3. 报告按严重程度和位置组织，不只输出总分。
4. 每条建议可以定位、忽略或创建修改任务。
5. 修复结果回到编辑稿，再次检查。
6. 用户决定是否正式保存。

## 8. 任务与错误恢复

### 8.1 用户可见任务信息

- 任务名称，例如“正在生成第 12 章”；
- 所属项目和章节；
- 当前阶段和百分比；
- 已保存的部分结果；
- 预计影响和可用操作；
- 失败后的重试、继续或接受部分结果。

任务 ID、线程、内部异常类型和原始模型响应默认隐藏。

### 8.2 错误类型与行动

| 错误类型 | 用户提示 | 可执行行动 |
| --- | --- | --- |
| 模型未配置 | 需要配置模型才能使用此功能 | 前往设置、稍后处理 |
| 认证失败 | 模型服务未接受当前凭据 | 重新填写并测试；不回显密钥 |
| 网络/超时 | 请求未在预期时间完成，已保留进度 | 继续、重试当前阶段、稍后恢复 |
| 解析失败 | 返回内容无法完整识别，原始候选已保留 | 重试解析、重新生成、查看高级详情 |
| 文件不可读 | 无法读取选定文件 | 重新选择、检查权限、跳过失败文件 |
| 数据不可写 | 项目无法保存，当前编辑仍在内存/临时区 | 选择新位置、重试保存、导出紧急备份 |
| 项目损坏 | 项目清单或数据版本不兼容 | 只读打开、从备份恢复、查看迁移报告 |
| 任务不存在 | 任务可能因重启中断 | 恢复已保存结果、重新启动任务 |

### 8.3 取消

- 取消是“停止后续处理”，不是删除已有结果。
- 取消前说明已完成阶段和将保留的内容。
- 取消后任务状态和部分结果可查看。
- 用户可以从最近完成阶段继续。

## 9. 首次使用体验

### 9.1 欢迎页内容

- NOVAL 的一句话定位；
- 本地优先与远程模型边界；
- “创建第一个项目”主按钮；
- “打开已有项目”和“试用原创示例”次要入口；
- 安装或环境异常时显示诊断入口。

欢迎页不显示功能宣传卡片、空白统计、内部版本数据或日志。

### 9.2 创建项目向导

| 步骤 | 必填 | 默认 |
| --- | --- | --- |
| 项目名称 | 是 | 无 |
| 项目类型 | 是 | 原创 |
| 项目说明 | 否 | 空 |
| 数据位置 | 否 | 应用默认位置 |
| 模型配置 | 生成前必填 | 继承全局默认 |
| 开始方式 | 是 | 空项目/导入/示例三选一 |

### 9.3 向导完成页

根据项目类型只给一个主建议：

- 续写项目：导入语料；
- 分析项目：扫描并开始分析；
- 原创项目：填写设定或创建总体构想；
- 已导入草稿：打开章节编辑器。

## 10. 自动保存与确认点

### 自动保存

- 项目创建向导进度；
- 总体构想编辑稿；
- 章节规划编辑；
- 当前章节编辑稿；
- 长任务输入和部分结果；
- 质检报告；
- 当前项目、章节和页面位置。

### 必须确认

- 迁移或复制旧项目；
- 覆盖已有章节规划；
- 接受总体构想为正式规划基线；
- 删除语料索引、候选稿、计划或正式章节；
- 用候选稿更新正式章节；
- 在严重质检问题未处理时正式保存；
- 项目归档、恢复和永久删除。

### 不应反复确认

- 普通页面切换；
- 自动保存；
- 查看分析或版本；
- 任务完成；
- 创建新的候选稿（不覆盖正式内容时）。

## 11. 流程验收场景

后续实现至少要自动化验证：

1. 新用户使用原创示例创建项目并进入项目概览。
2. 两个项目之间切换，语料、任务、候选稿和正式章节不混淆。
3. 页面刷新后恢复当前项目、章节和进行中任务。
4. 生成中途超时，部分正文仍可恢复并继续。
5. 修改失败，原正文和版本仍存在。
6. 质检失败，不影响编辑稿和正式章节。
7. 正式保存原子失败，旧正式版本保持完整。
8. 旧 `longzu` 项目通过兼容层打开，不移动原始语料。
9. 未配置模型时仍可浏览项目和编辑文本。
10. 普通用户主流程中不出现 API、JSON、task ID 或内部路径。
