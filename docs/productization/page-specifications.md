# NOVAL Phase 1 页面职责规格

状态：待用户确认  
依赖：[02-information-architecture.md](02-information-architecture.md)、[user-flows.md](user-flows.md)

## 1. 通用页面规则

1. 每页只能有一个视觉优先级最高的主操作。
2. 项目页固定显示当前项目；章节工作区固定显示当前章节。
3. 空状态必须提供可执行下一步，不展示没有意义的 0 统计矩阵。
4. 加载状态区分短请求和长任务；长任务展示阶段、进度和恢复入口。
5. 错误状态不显示堆栈、task ID、JSON 或完整路径。
6. 成功状态说明保存到哪个产品对象，例如“已保存为候选稿”，而不是只显示“成功”。
7. 文件路径可在“查看数据位置”中提供，不作为页面主信息。
8. 删除、覆盖、迁移和正式保存遵循版本保护和确认规则。

## 2. 首页

| 项目 | 规格 |
| --- | --- |
| 页面名称 | 首页 |
| 页面目的 | 帮助用户开始、继续或恢复最近的工作 |
| 主要用户 | 所有用户，尤其是首次使用和跨项目用户 |
| 核心信息 | 欢迎/最近项目、继续创作、进行中任务、需要处理的问题、应用可用状态 |
| 主要操作 | 首次用户“创建第一个项目”；已有用户“继续上次工作” |
| 次要操作 | 打开项目、查看全部项目、恢复任务、打开诊断 |
| 空状态 | 欢迎说明 + 创建项目 + 打开已有项目 + 原创示例；不显示空白统计 |
| 加载状态 | 最近项目和任务使用骨架；主创建入口始终可用 |
| 错误状态 | 项目索引不可读时提供重新扫描、打开项目目录或诊断；不阻塞创建新项目 |
| 成功状态 | 显示最近项目、明确下一步和已恢复任务 |
| 下一步行动 | 进入项目概览或完成首次向导 |
| 不应放置 | 单项目章节统计、语料全文、模型高级参数、技术日志、固定作品名称 |

## 3. 项目列表

| 项目 | 规格 |
| --- | --- |
| 页面名称 | 项目 |
| 页面目的 | 创建、查找、打开、归档、备份和恢复本地项目 |
| 主要用户 | 所有管理多个作品的用户 |
| 核心信息 | 项目名称、类型、最近打开、准备状态、创作进度、数据健康状态 |

| 次要操作 | 搜索/筛选、打开、备份、导入已有项目、归档、查看数据位置 |

## 4. 项目详情（项目概览）

| 项目 | 规格 |
| --- | --- |
| 页面名称 | 项目概览 |
| 页面目的 | 汇总当前项目的准备度、创作进度和推荐下一步 |
| 主要用户 | 项目创建者和持续创作者 |
| 核心信息 | 项目名称/类型、模型状态、语料状态、分析状态、构想/规划状态、章节进度、最近活动 |
| 主要操作 | 根据状态动态显示一个“继续下一步”按钮 |
| 次要操作 | 编辑项目资料、打开最近章节、查看任务、备份项目 |
| 空状态 | 新项目显示准备清单，不显示 0/0 仪表盘 |
| 加载状态 | 各状态摘要独立加载；继续按钮仅在目标状态确认后启用 |
| 错误状态 | 某模块不可读时标注受影响范围，允许进入未受影响模块 |
| 成功状态 | 显示完成项、当前阶段和下一目标，例如“总体构想已确认” |
| 下一步行动 | 导入语料、开始分析、创建构想、继续当前章节或导出 |
| 不应放置 | 全量章节正文、所有配置字段、原始导入报告、详细任务日志 |

## 5. 语料库

| 项目 | 规格 |
| --- | --- |
| 页面名称 | 语料库 |
| 页面目的 | 导入、扫描、校验并浏览当前项目的合法只读语料 |
| 主要用户 | 续写用户、分析用户、高级用户 |
| 核心信息 | 来源、文件/章节数量、扫描状态、异常、章节列表、选中章节详情、只读标识 |
| 主要操作 | 无语料时“导入语料”；已有未扫描内容时“扫描章节” |
| 次要操作 | 上传文件、添加目录引用、筛选章节、查看导入报告、重新扫描、移除索引 |
| 空状态 | 解释支持格式、版权责任和只读原则；提供导入与原创项目跳过入口 |
| 加载状态 | 文件解析显示当前文件/章节阶段；章节列表可逐步出现 |
| 错误状态 | 按文件列出失败原因；已成功章节保留；允许重选、跳过和重试 |
| 成功状态 | 显示新增、跳过、异常和总章节数，并给出“开始分析” |
| 下一步行动 | 分析中心；原创项目可直接进入总体构想 |
| 不应放置 | 固定 8 个文件/333 章、特定作品路径、知识库编辑、章节生成 |

## 6. 分析中心

| 项目 | 规格 |
| --- | --- |
| 页面名称 | 分析中心 |
| 页面目的 | 管理和浏览从项目语料得到的风格与知识结果 |
| 主要用户 | 分析用户、续写用户、需要一致性上下文的创作者 |
| 核心信息 | 分析覆盖范围、更新时间、过期状态、风格摘要、人物/地点/主题/情节知识入口 |
| 主要操作 | “开始分析”或“更新受影响分析” |
| 次要操作 | 选择章节范围、查看风格维度、浏览知识库、重试失败章节、查看来源 |
| 空状态 | 无语料时引导到语料库；有语料时说明可分析内容和调用影响 |
| 加载状态 | 展示分析章节数、当前阶段、成功/失败计数和取消；已有结果仍可浏览 |
| 错误状态 | 失败章节单独列出；保留成功结果；提供重试失败项和配置模型入口 |
| 成功状态 | 展示覆盖率、生成时间、主要发现和“用于创作”状态 |
| 下一步行动 | 总体构想或返回语料修复异常 |
| 不应放置 | 正文编辑器、正式保存、模型 token 参数、原始模型 JSON |

## 7. 自动构想（总体构想）

| 项目 | 规格 |
| --- | --- |
| 页面名称 | 总体构想 |
| 页面目的 | 生成、编辑、比较并确认一部作品的整体方向 |
| 主要用户 | 续写用户、原创长篇规划用户 |
| 核心信息 | 用户方向、使用的上下文、构想候选、核心冲突、角色弧线、结构、确认状态 |
| 主要操作 | 无构想时“生成总体构想”；有候选时“确认总体构想” |
| 次要操作 | 手工创建、保存编辑、重新生成、比较版本、修改方向、查看使用的知识 |
| 空状态 | 提供可选方向输入和上下文准备提示；允许无语料原创构想 |
| 加载状态 | 显示分析上下文准备、生成、解析、保存阶段；方向输入已保存 |
| 错误状态 | 保留方向和旧构想；提供重试、手工编写和查看模型连接 |
| 成功状态 | 候选可编辑；确认后显示确认时间和“生成章节规划” |
| 下一步行动 | 章节规划 |
| 不应放置 | 固定“下一部/第六部”文案、正式章节全文、文件保存路径、原始 Prompt |

## 8. 章节规划

| 项目 | 规格 |
| --- | --- |
| 页面名称 | 章节规划 |
| 页面目的 | 建立和维护全书章节结构，并选择当前创作章节 |
| 主要用户 | 长篇规划用户、续写用户 |
| 核心信息 | 总体构想摘要、章节顺序、标题、目标、情节节点、角色、状态、依赖和完成度 |
| 主要操作 | 无规划时“生成完整章节规划”；有规划时“开始下一未完成章节” |
| 次要操作 | 新建/编辑/重排章节、重新生成选中范围、筛选状态、查看前后章、批量检查 |
| 空状态 | 无已确认构想时引导确认构想；也允许手工创建第一章计划 |
| 加载状态 | 分批显示已生成计划和当前批次；允许查看已完成部分 |
| 错误状态 | 保留旧规划和已完成批次；提供重试失败批次或手工补全 |
| 成功状态 | 显示覆盖率、异常依赖和推荐下一章 |
| 下一步行动 | 进入选中章节的章节工作区 |
| 不应放置 | 整章正文编辑、全局模型配置、临时生成历史、底层 JSON 文件 |

## 9. 章节生成

章节生成是“章节工作区”的生成状态，不是一级导航。

| 项目 | 规格 |
| --- | --- |
| 页面名称 | 章节工作区 / 生成 |
| 页面目的 | 根据当前章节计划生成可恢复的完整候选稿 |
| 主要用户 | 续写用户、原创创作者 |
| 核心信息 | 当前项目/章节、计划摘要、前后文、目标字数、补充要求、生成进度、部分正文 |
| 主要操作 | “生成完整章节”或任务中“继续生成” |
| 次要操作 | 调整要求、取消、接受部分稿、重新生成当前段、保存候选稿 |
| 空状态 | 无计划时引导创建/选择章节计划；允许切换到手工编辑 |
| 加载状态 | 分段展示阶段、字数和已保存部分；用户可离开页面 |
| 错误状态 | 保留已生成部分和输入；提供继续、重试当前阶段或接受部分稿 |
| 成功状态 | 生成候选稿进入编辑器，同时显示规则检查摘要 |
| 下一步行动 | 人工编辑和审核 |
| 不应放置 | task ID、生成协议选择、token 参数、其他项目任务、直接正式覆盖 |

## 10. 章节编辑

| 项目 | 规格 |
| --- | --- |
| 页面名称 | 章节工作区 / 编辑 |
| 页面目的 | 人工编辑当前章节，并安全应用 AI 修改候选 |
| 主要用户 | 所有创作者，尤其是已有草稿用户 |
| 核心信息 | 当前编辑稿、自动保存状态、字数、计划对照、版本基点、未采纳候选 |
| 主要操作 | “保存候选稿”；满足条件时“进入质检” |
| 次要操作 | 局部修改、整章重写、继续写作、比较差异、撤销到版本、手工检查 |
| 空状态 | 新章节提供手工开始和生成章节两个入口；不自动生成 |
| 加载状态 | 打开大章节时保持框架和版本信息；编辑器就绪前不可覆盖内容 |
| 错误状态 | 自动保存失败时持续保留本地编辑并提供紧急导出；AI 修改失败不影响正文 |
| 成功状态 | 显示最近保存时间、版本和候选稿状态 |
| 下一步行动 | AI 质检或正式保存确认 |
| 不应放置 | 其他章节全文列表、项目路径、原始模型响应、无确认的自动覆盖 |

## 11. AI 质检

| 项目 | 规格 |
| --- | --- |
| 页面名称 | 章节工作区 / AI 质检 |
| 页面目的 | 检查当前编辑稿的完整性、一致性和质量，并把建议转成可控修改 |
| 主要用户 | 准备正式保存章节的创作者 |
| 核心信息 | 规则检查、严重程度、问题位置、计划节点覆盖、AI 建议、已忽略/已修复状态 |
| 主要操作 | “开始深度质检”；报告完成后“处理严重问题”或“确认进入正式保存” |
| 次要操作 | 重跑规则检查、定位正文、忽略建议、创建修改任务、导出报告 |
| 空状态 | 先自动运行规则检查；模型未配置时仍提供本地检查 |
| 加载状态 | 分开显示本地规则完成和 AI 分析进度；正文保持可查看 |
| 错误状态 | AI 失败时保留规则结果和正文；允许重试或跳过并记录原因 |
| 成功状态 | 展示已通过项、待处理项和正式保存风险摘要 |
| 下一步行动 | 返回编辑修复或确认正式保存 |
| 不应放置 | 自动替用户接受修改、只给总分无证据、错误堆栈、跨项目报告 |

## 12. 临时记录（候选稿）

| 项目 | 规格 |
| --- | --- |
| 页面名称 | 章节库 / 候选稿 |
| 页面目的 | 管理尚未正式确认的生成、修改和导入草稿 |
| 主要用户 | 需要比较方案、恢复未完成工作的创作者 |
| 核心信息 | 所属章节、来源、创建时间、字数、任务状态、质检状态、最近编辑 |
| 主要操作 | 打开候选稿继续编辑 |
| 次要操作 | 比较、重命名、复制、删除到回收站、关联章节、导出 |
| 空状态 | 说明候选稿由生成或编辑产生，并提供“进入创作” |
| 加载状态 | 列表骨架；筛选和项目上下文保持可见 |
| 错误状态 | 单项不可读时隔离并提供恢复/诊断，不影响其他候选稿 |
| 成功状态 | 保存新候选后高亮并说明尚未进入正式章节库 |
| 下一步行动 | 章节编辑、质检或正式保存 |
| 不应放置 | 与正式章节混合排序、内部 generation ID、硬删除无确认、所有任务日志 |

## 13. 正式章节库

| 项目 | 规格 |
| --- | --- |
| 页面名称 | 章节库 / 正式章节 |
| 页面目的 | 管理已确认章节、版本、顺序、状态和导出 |
| 主要用户 | 所有持续创作和交付用户 |
| 核心信息 | 章节顺序、标题、字数、正式版本、更新时间、质检摘要、导出状态 |
| 主要操作 | 打开下一章或当前章节详情；完成作品时“导出项目” |
| 次要操作 | 查看版本、创建修订候选、单章导出、批量导出、恢复旧版本、归档 |
| 空状态 | 说明正式章节需要用户确认，提供打开候选稿或开始第一章 |
| 加载状态 | 列表和统计渐进加载；不阻塞打开已缓存章节 |
| 错误状态 | 某章损坏时标记并提供只读恢复；不重排或删除其他章节 |
| 成功状态 | 正式保存后显示版本号、备份状态和下一章入口 |
| 下一步行动 | 下一章、修订或导出 |
| 不应放置 | 未确认候选稿混入正式正文、直接覆盖无版本、技术文件路径作为标题 |

## 14. 设置

| 项目 | 规格 |
| --- | --- |
| 页面名称 | 设置 |
| 页面目的 | 配置模型、数据、隐私、备份和高级诊断 |
| 主要用户 | 所有用户；高级区面向技术用户 |
| 核心信息 | 模型连接状态、默认模型、数据位置、备份状态、隐私说明、应用版本 |
| 主要操作 | 当前有阻塞时“修复并测试连接”；正常时“保存设置” |
| 次要操作 | 项目覆盖配置、选择数据目录、创建备份、恢复、查看诊断、高级参数 |
| 空状态 | 无模型配置时提供受支持字段和安全说明；API Key 使用密码输入且不可回显 |
| 加载状态 | 读取本地配置时不显示密钥值；连接测试独立展示进度 |
| 错误状态 | 按认证、网络、模型、目录权限分类；提供修复步骤和安全详情 |
| 成功状态 | 显示“连接可用”、模型、测试时间和配置作用域，不显示密钥 |
| 下一步行动 | 返回触发设置需求的原页面 |
| 不应放置 | 明文 API Key、普通用户默认展开的 token/CORS/端口、项目正文、原始堆栈 |

## 16. 跨页面状态要求

### 16.1 固定应用框架

- 全局导航；
- 当前项目切换器；
- 全局任务状态；
- 模型/数据阻塞提示；
- 页面内容区。

### 16.2 必须持久化的前端状态

- 当前项目；
- 当前章节；
- 当前创作步骤；
- 未保存编辑稿和自动保存版本；
- 运行中任务及所属项目/章节；
- 用户选择的筛选和排序（低优先级）。

### 16.3 不应只存在前端内存的状态

- 正文草稿；
- 已确认总体构想；
- 章节规划；
- 任务的恢复信息；
- 正式保存结果；
- 迁移、备份和恢复状态。

## 17. 页面规格验收

进入 UI 实现阶段前，原型或实现应验证：

1. 首次用户看不到空白统计和特定作品名称。
2. 任意项目页面可以识别并切换当前项目。
3. 章节工作区可以识别当前章节和稳定版本。
4. 每页主操作不超过一个。
5. 空、加载、错误、成功四类状态都有设计。
6. 任务在页面切换后继续，并可从全局入口找回。
7. 候选稿和正式章节不能视觉混淆。
8. 普通流程不出现 API、JSON、task ID、端口和内部路径。
9. 窄屏下导航和章节工作区不发生内容遮挡。
10. 所有覆盖和删除操作符合版本与恢复规则。
