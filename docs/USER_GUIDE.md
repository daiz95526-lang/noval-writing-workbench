# 用户使用指南

NOVAL 是本地优先的 AI 写作工作台。它把语料、分析缓存、草稿、生成记录和正式章节保存在本机文件系统中。

## 推荐工作流

1. 准备合法语料：把你有权处理的文本放入本地数据目录。
2. 导入语料：在「语料管理」中扫描本地文本，确认章节数量和统计信息。
3. 分析风格：选择章节进行风格分析，必要时构建全局风格知识。
4. 构建知识库：生成角色、设定、情节、主题和风格规则。
5. 自动构想下一部：基于锚点章节生成总体规划。
6. 完整章节规划：接受或修订总体规划后生成章节计划。
7. 生成章节：按章节计划生成正文，可使用分段生成和完整性检查。
8. 深度质检：对生成章节执行 AI 审稿，按建议修复。
9. 保存与导出：满意后保存到正式章节库，并导出 Markdown 或 TXT。

## 数据类型

- 原始语料：用户自行准备的文本、电子书或转换结果。
- 处理后章节：系统识别和切分后的章节文本。
- 风格缓存：章节风格分析结果。
- 知识库：角色、世界观、情节节点、主题和风格规则。
- 临时生成记录：每次模型生成的临时结果。
- 草稿与版本：编辑区保存的草稿和历史快照。
- 正式章节：确认进入作品库的章节。
- 导出文件：用于外部编辑或备份的 Markdown/TXT。

## 使用边界

请只导入你有权处理的文本。公开仓库、分享问题反馈或提交 Pull Request 时，不要附带原文、生成正文、私有规划、分析缓存或真实 API Key。

## 错误恢复

长任务页面会显示任务状态、进度、日志、失败原因和部分成功结果。生成任务如果部分成功，优先保存可恢复的正文，再决定是否重新生成后续段落。

本项目目前以本地文件作为存储层。重要写作项目建议定期备份 `writing_projects/` 和你配置的数据目录。
